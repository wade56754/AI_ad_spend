创建一个广告投手消耗管理系统前端，使用 Next.js 14 App Router + TypeScript + Tailwind CSS。

后端 API 基础 URL: http://localhost:8000/api
所有 API 响应格式统一为: {"data": ..., "error": null, "meta": ...}

需要创建的页面：

1. 投手消耗上报页面 (/report/spend)
   - 表单字段：日期选择器、项目下拉、国家下拉、投手ID输入、平台输入、金额输入(USDT)、备注文本域
   - 必填字段：日期、项目、投手ID、金额
   - 提交到 POST /api/ad-spend
   - 显示成功/失败提示
   - 提交成功后重置表单

2. 财务收支录入页面 (/finance/ledger)
   - 表单字段：日期、收支类型选择(收入/支出)、金额、币种选择(USDT/CNY)、账户、手续费、项目ID(可选)、投手ID(可选)、备注
   - 必填字段：日期、收支类型、金额、币种
   - 提交到 POST /api/ledger
   - 表单验证和错误提示

3. 对账结果页面 (/reconcile)
   - 表格展示对账结果，包含列：投手名称、项目名称、消耗日期、消耗金额、财务日期、财务金额、匹配度、状态、原因
   - 状态筛选下拉框（全部/已匹配/待审核）
   - 待审核记录(status=need_review)显示"确认匹配"按钮
   - 调用 GET /api/reconcile?status=xxx 获取数据
   - 点击确认调用 PATCH /api/reconcile/{id} 更新状态为matched
   - 刷新按钮

4. 分析页面 (/analytics)
   - 月度报表生成：年份和月份选择，提交到 POST /api/reports/monthly
   - 诊断报告查看：年份和月份选择，调用 GET /api/reports/diagnostic?year=xxx&month=xxx&format=text
   - 显示报表结果和诊断报告文本

5. 设置页面 (/settings)
   - 项目管理：表格展示，支持增删改查
   - 投手管理：表格展示，支持增删改查

设计要求：
- 使用 Tailwind CSS 现代化设计
- 响应式布局，支持移动端
- 统一的错误处理和提示样式
- 加载状态显示（按钮禁用、加载动画）
- 完整的表单验证和实时反馈
- 美观的表格样式
- 统一的颜色方案（蓝色主题）

技术栈：
- Next.js 14 (App Router)
- TypeScript
- Tailwind CSS
- 使用 fetch API 调用后端，统一错误处理

API 端点详情：
- POST /api/ad-spend - 创建消耗上报
- GET /api/ad-spend - 查询消耗列表
- POST /api/ledger - 创建财务记录
- GET /api/ledger - 查询财务列表
- POST /api/reconcile/run - 执行对账
- GET /api/reconcile - 查询对账结果
- PATCH /api/reconcile/{id} - 更新对账状态
- POST /api/reports/monthly - 生成月度报表
- GET /api/reports/diagnostic - 获取诊断报告

所有 API 请求使用 fetch，设置 Content-Type: application/json
响应格式统一为 {"data": ..., "error": null, "meta": ...}
错误时显示 result.error，成功时显示 result.data
